API Reference

Codebook

TurboQuant.LloydMaxCodebook — Type
LloydMaxCodebook

Stores precomputed optimal centroids and decision boundaries for a given bit-width and target distribution (Beta distribution of rotated coordinates).

source
TurboQuant.solve_codebook — Function
solve_codebook(d, b; max_iter=1000, tol=1e-10, n_quad=10000)

Solve the Lloyd-Max optimal quantizer for the Beta distribution arising from random rotation in dimension d, at bit-width b.

Returns a LloydMaxCodebook with 2^b centroids and 2^b-1 decision boundaries.

source
TurboQuant.quantize_scalar — Function
quantize_scalar(val, cb::LloydMaxCodebook)

Quantize a single scalar value using the precomputed codebook. Returns the index (1-based) of the nearest centroid.

source
TurboQuant.dequantize_scalar — Function
dequantize_scalar(idx, cb::LloydMaxCodebook)

Look up the centroid value for index idx.

source
TurboQuant.precompute_codebooks — Function
precompute_codebooks(d; max_bits=8)

Precompute Lloyd-Max codebooks for bit-widths 1 through max_bits. Returns a Dict{Int, LloydMaxCodebook}.

source

Rotation

TurboQuant.RandomRotation — Type
RandomRotation

Dense random orthogonal rotation matrix generated via QR decomposition of a Gaussian random matrix. O(d²) storage and O(d²) multiply.

source
TurboQuant.HadamardRotation — Type
HadamardRotation

Structured random rotation using randomized Hadamard transform: Π = D₃ H D₂ H D₁ (three rounds of Hadamard with random diagonal signs).

This achieves O(d log d) rotation instead of O(d²), with similar distributional properties for sub-Gaussian tail bounds.

source
TurboQuant.rotate — Function
rotate(R::RandomRotation, x)

Apply forward rotation: y = Π · x. Supports vectors (d,) and matrices (d, N) for batch processing.

source
rotate(R::HadamardRotation, x::AbstractVector)

Apply structured Hadamard rotation to a vector.

source
rotate(R::HadamardRotation, X::AbstractMatrix)

Batch rotation: each column is a vector to rotate.

source
TurboQuant.rotate_back — Function
rotate_back(R::RandomRotation, y)

Apply inverse rotation: x = Π^T · y.

source
rotate_back(R::HadamardRotation, y::AbstractVector)

Apply inverse structured Hadamard rotation. The inverse of Di H / √d is H Di / √d (since H and D are self-inverse up to scaling).

source

MSE Quantizer

TurboQuant.TurboQuantMSE — Type
TurboQuantMSE{R}

MSE-optimal vector quantizer using:

  1. Random rotation Π to normalize coordinate distribution
  2. Per-coordinate Lloyd-Max scalar quantization
  3. Norm preservation via separate float storage

Type parameter R is the rotation type (RandomRotation or HadamardRotation).

source

CompressedVectorMSE

Compressed representation of a batch of vectors using MSE quantization.

source

Initialize the MSE quantizer for vectors of dimension d at bit-width b.

quantize(tq::TurboQuantMSE, X::AbstractMatrix)

Quantize a batch of vectors. X is (d, N) where each column is a vector. Returns a CompressedVectorMSE.

TurboQuant.dequantize — Method
dequantize(tq::TurboQuantMSE, comp::CompressedVectorMSE)

Dequantize a compressed representation back to approximate vectors. Returns a (d, N) matrix.

Inner Product Quantizer

TurboQuant.TurboQuantProd — Type
TurboQuantProd{R}

Unbiased inner product quantizer using two-stage pipeline:

  1. TurboQuant_MSE at (b-1) bits for base approximation
  2. QJL (Quantized Johnson-Lindenstrauss) 1-bit projection on residual

Guarantees E[⟨y, x̃⟩] = ⟨y, x⟩ (unbiased inner product estimation).

source
TurboQuant.CompressedVectorProd — Type
CompressedVectorProd

Compressed representation using the two-stage prod quantizer.

source
TurboQuant.setup — Method
setup(::Type{TurboQuantProd}, d, b; seed=UInt64(42), seed_qjl=UInt64(123), use_hadamard=false)

Initialize the unbiased inner product quantizer. Uses (b-1) bits for MSE stage + 1 bit for QJL residual stage. Requires b ≥ 2.

source
TurboQuant.quantize — Method
quantize(tq::TurboQuantProd, X::AbstractMatrix)

Two-stage quantization:

  1. MSE quantize at (b-1) bits
  2. Compute residual, apply QJL (sign of random projection)
source
TurboQuant.dequantize — Method
dequantize(tq::TurboQuantProd, comp::CompressedVectorProd)

Two-stage dequantization: x̃ = x̃mse + √(π/2)/d · ‖r‖ · S^T · qjlsigns

source
TurboQuant.inner_product — Function
inner_product(tq::TurboQuantProd, comp::CompressedVectorProd, y::AbstractVector)

Compute approximate inner products ⟨x̃_i, y⟩ for all quantized vectors, without full dequantization. Returns a vector of length N.

This is unbiased: E[result] = ⟨x, y⟩ for each vector.

source
TurboQuant.inner_product_bias — Function
inner_product_bias(tq::TurboQuantProd, X::AbstractMatrix, y::AbstractVector; n_trials=100)

Estimate the bias of inner product estimation by repeated quantization. Returns (meanbias, stdbias) where bias = estimatedip - trueip.

source

KV Cache

TurboQuant.KVCache — Type
KVCache{R}

Quantized KV cache for transformer attention. Supports:

  • Online streaming quantization (no calibration needed)
  • Mixed-precision: outlier channels get extra bits
  • On-the-fly dequantization during attention computation
source
TurboQuant.compress_kv! — Function
compress_kv!(cache::KVCache, K::AbstractMatrix, V::AbstractMatrix)

Add a new token's K and V to the cache with online quantization. K, V are (headdim, nheads) — one vector per attention head.

source
TurboQuant.attention_with_quantized_kv — Function
attention_with_quantized_kv(cache::KVCache, Q::AbstractMatrix;
                             scale=nothing)

Compute attention output using quantized KV cache. Q is (headdim, nheads) for the current query token.

Computes: output = softmax(Q^T K / √d) V using quantized K and V. Returns (headdim, nheads) output.

source
TurboQuant.memory_usage — Function
memory_usage(cache::KVCache)

Estimate memory usage of the quantized KV cache in bytes.

source
TurboQuant.fp16_memory_usage — Function
fp16_memory_usage(cache::KVCache)

Memory usage if KV cache were stored in Float16.

Nearest Neighbor Search

TurboQuant.TurboQuantIndex — Type
TurboQuantIndex{R}

A nearest neighbor search index using TurboQuant compression. No codebook learning required — vectors are quantized online using random rotation + Lloyd-Max quantization.

source
TurboQuant.build_index — Function
build_index(X::AbstractMatrix, bit_width::Int;
            seed=UInt64(42), use_hadamard=true, batch_size=1000)

Build a nearest neighbor search index from database vectors. X is (d, N) where each column is a database vector.

This is essentially instantaneous compared to PQ (no k-means training).

source
TurboQuant.search — Method
search(index::TurboQuantIndex, query::AbstractVector, k::Int)

Find the k approximate nearest neighbors of query in the index. Returns (indices, scores) where indices are 1-based into the original database and scores are approximate inner products (higher = more similar).

source
TurboQuant.batch_search — Function
batch_search(index::TurboQuantIndex, queries::AbstractMatrix, k::Int)

Search for k nearest neighbors for each query column in queries. Returns (indicesmatrix, scoresmatrix) each of size (k, n_queries).

source
TurboQuant.recall_at_k — Function
recall_at_k(index::TurboQuantIndex, queries::AbstractMatrix,
            ground_truth::AbstractMatrix, k::Int)

Compute recall@1@k: fraction of queries where the true nearest neighbor is among the top-k results from the approximate search.

ground_truth is (1, n_queries) containing the index of each query's true NN.
